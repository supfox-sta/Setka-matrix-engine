# Сетка Matrix

Проект «Сетка Matrix» разворачивает собственную инстанцию Matrix на домене `setka-matrix.ru`. Сервер построен на базе открытого Synapse и предназначен для безопасного пространства общения внутри вашей локальной сети/сообщества.

## Что внутри
- **Домен:** `setka-matrix.ru`
- **Платформа:** Synapse (Matrix homeserver)
- **Цель:** собственная федеративная инстанция Matrix с контролем над инфраструктурой и персональными настройками модерации.

## Начало работы
1. Убедитесь, что установлен Python 3.13 и все зависимости Synapse.
2. Пропишите настройки в `homeserver.yaml`, указав `server_name: setka-matrix.ru`, конфигурацию баз данных и TLS.
3. Создайте пользователя администратора командой:
   ```
   register_new_matrix_user -c homeserver.yaml https://localhost:8008
   ```
4. Запустите Synapse и убедитесь, что веб-интерфейс и Federation работают:
   ```
   python -m synapse.app.homeserver --config-path homeserver.yaml
   ```

## Развертывание и поддержка
- Настраивайте обратный прокси (nginx, Caddy) для TLS и проксирования портов 8448/8008.
- Подключайте модули и плагины Synapse при помощи `module_config`, если нужны кастомные авторизации, антиспам или интеграции.
- Используйте `synapse_admin` или Matrix SDK для автоматизации задач (создание комнат, розетка пользователей и т.д.).

## Лицензия
Поскольку Synapse распространяется под GNU AGPLv3, и Сетка Matrix расширяет его функциональность, рекомендуется придерживаться той же лицензии (AGPL-3.0-only). Это сохраняет свободу кода и гарантирует, что все улучшения остаются доступными сообществу при сетевой федерации.

Если нужна более либеральная лицензия для публичных библиотек/веб-интерфейсов, можно рассмотреть Apache 2.0 или MIT для отдельных компонентов, но ядро сервера лучше оставлять под AGPLv3.
